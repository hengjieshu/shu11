# 影刀 API 接口说明

本文档说明如何通过影刀 RPA 自动更新"实时销售"表格数据。

---

## API 基础地址

```
http://你的服务器IP:3001
```

- 本地开发：`http://localhost:3001`
- Mac mini 部署后：`http://Mac mini的IP地址:3001`

---

## 接口列表

### 1. 添加单条销售记录

**接口地址**：`POST /api/yingdao/sale`

**请求示例**：
```json
{
  "商品名称": "iPhone 15 Pro",
  "数量": 2,
  "单价": 8999,
  "总价": 17998,
  "销售渠道": "淘宝",
  "销售时间": "2026-03-25",
  "订单状态": "已付款"
}
```

**字段说明**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| 商品名称 | 文本 | 商品名称 |
| 数量 | 数字 | 销售数量 |
| 单价 | 数字 | 单价金额 |
| 总价 | 数字 | 总价金额 |
| 销售渠道 | 选择 | 淘宝/京东/拼多多/抖音/线下/其他 |
| 销售时间 | 日期 | 格式：YYYY-MM-DD |
| 订单状态 | 选择 | 待付款/已付款/已发货/已完成/已取消 |

**响应示例**：
```json
{
  "success": true,
  "rowId": 1,
  "message": "销售记录添加成功"
}
```

### 2. 批量导入销售记录

**接口地址**：`POST /api/yingdao/sales`

**请求示例**：
```json
{
  "records": [
    {
      "商品名称": "iPhone 15 Pro",
      "数量": 2,
      "单价": 8999,
      "总价": 17998,
      "销售渠道": "淘宝",
      "销售时间": "2026-03-25",
      "订单状态": "已付款"
    },
    {
      "商品名称": "MacBook Pro",
      "数量": 1,
      "单价": 14999,
      "总价": 14999,
      "销售渠道": "京东",
      "销售时间": "2026-03-25",
      "订单状态": "已发货"
    }
  ]
}
```

**响应示例**：
```json
{
  "success": true,
  "total": 2,
  "imported": 2,
  "failed": 0,
  "results": [...]
}
```

---

### 3. 清空销售数据

**接口地址**：`DELETE /api/yingdao/sales`

**响应示例**：
```json
{
  "success": true,
  "message": "已清空 10 条销售记录"
}
```

---

### 4. 获取字段列表

**接口地址**：`GET /api/yingdao/fields`

**响应示例**：
```json
[
  {"id": 1, "name": "商品名称", "type": "text"},
  {"id": 2, "name": "数量", "type": "number"},
  ...
]
```

## 影刀配置步骤

### 方法一：使用「HTTP 请求」组件

1. 在影刀流程中添加 **「HTTP 请求」** 组件
2. 配置如下：

| 配置项 | 值 |
|--------|-----|
| 请求方式 | POST |
| URL | `http://你的IP:3001/api/yingdao/sale` |
| Content-Type | application/json |
| Body | 见下方示例 |

**Body 示例（使用变量）**：
```json
{
  "商品名称": "{{商品名称变量}}",
  "数量": {{数量变量}},
  "单价": {{单价变量}},
  "总价": {{总价变量}},
  "销售渠道": "{{渠道变量}}",
  "销售时间": "{{时间变量}}",
  "订单状态": "{{状态变量}}"
}
```

### 方法二：使用 Python 脚本组件

```python
import requests
import json

# API 地址（部署后改为实际 IP）
api_url = "http://localhost:3001/api/yingdao/sale"

# 销售数据
data = {
    "商品名称": 商品名称,      # 影刀变量
    "数量": 数量,             # 影刀变量
    "单价": 单价,             # 影刀变量
    "总价": 总价,             # 影刀变量
    "销售渠道": 销售渠道,     # 影刀变量
    "销售时间": 销售时间,     # 影刀变量
    "订单状态": 订单状态      # 影刀变量
}

# 发送请求
response = requests.post(api_url, json=data)
result = response.json()

# 返回结果
if result.get("success"):
    print(f"成功添加记录，ID: {result['rowId']}")
else:
    print(f"添加失败: {result.get('error')}")
```

## 常见使用场景

### 场景1：电商平台订单同步

1. 影刀定时抓取淘宝/京东订单
2. 解析订单数据
3. 调用 API 批量导入

### 场景2：Excel 数据导入

1. 影刀读取 Excel 文件
2. 遍历每行数据
3. 调用 API 添加记录

### 场景3：实时监控更新

1. 影刀监控文件夹/邮件
2. 收到新订单通知
3. 解析并调用 API 添加

---

## 注意事项

1. 确保 Mac mini 的 3001 端口可访问
2. 如需公网访问，建议配置 Nginx 反向代理和 HTTPS
3. 可根据需要添加 API 认证（API Key）
